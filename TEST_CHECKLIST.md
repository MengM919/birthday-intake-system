# 六步商业化重构验收清单

## 顾客端

- [ ] 裸链接只显示“请从商家发来的专属链接开始”，不能上传或提交。
- [ ] 专属 `?order=...&token=...` 链接可领取订单。
- [ ] 顾客无法修改已锁定套餐。
- [ ] 顾客可在 `claimed`、`draft`、`needs_revision` 状态切换 11 套模板。
- [ ] `submitted` 后模板选择被锁定。
- [ ] 顾客流程只有 6 步，顶部显示 Step X / 6。
- [ ] 未完成步骤不能跳过；已完成步骤可以返回。
- [ ] Step 1 联系方式、模板必填。
- [ ] Step 2 昵称、生日、关系必填；非匿名送礼人必须填昵称。
- [ ] Step 3 必须有封面和至少一张相册照。
- [ ] 基础心意款超过 12 张时被阻止。
- [ ] 惊喜升级款超过 30 张时被阻止。
- [ ] 单张上传失败可重试，其他照片不受影响。
- [ ] HEIC/HEIF 有可操作的转换或重试提示。
- [ ] 重点照片最多 8 张，排序/封面焦点保存成功。
- [ ] Step 4 主祝福必填且不超过 15 字。
- [ ] 文案助手只显示“我想亲手写 / 帮我找找灵感 / 帮我润色”，不宣称未接入的 AI。
- [ ] Step 5 基础款只显示固定模块说明。
- [ ] 升级款模块显示“已包含、可按需开启、不额外收费”。
- [ ] 开启许愿瓶、未来信箱、惊喜盲盒时，必填项被正确校验。
- [ ] 惊喜盲盒显示 8 种意象，内部保存 `surpriseBox` 和 `imageryCode`。
- [ ] Step 6 隐私授权默认未勾选。
- [ ] 顾客按钮文本为“确认提交制作资料”，顾客看不到 JSON。
- [ ] 提交成功提示订单号与后续制作说明。

## 已发布生日页

- [ ] 使用选中的 T01-T11 风格、配色与装饰，而非统一页面皮肤。
- [ ] 顶部功能按钮可滚动，点击后定位到对应模块。
- [ ] 倒计时显示“距离下一次生日还有多久”并带生日氛围装饰。
- [ ] 封面图采用保存的焦点，不遮挡寿星姓名或主祝福。
- [ ] 相册首组最多显示 8 张重点回忆。
- [ ] 其余照片可用“上一组 / 下一组”在同一区域分页浏览。
- [ ] 许愿瓶能打开输入窗口。
- [ ] 未来信箱会在设定日期前保持封存。
- [ ] 惊喜盲盒可打开、可关闭、再次打开，并在低动效设置下正常。
- [ ] 背景音乐只在首次触摸/点击页面后播放，浏览器自动播放限制下不会报错。
- [ ] 今日好运只有真实函数返回数据才显示内容。
- [ ] 一个数据源失败时仍展示另一个；两个失败时显示“今日好运暂时在路上，请稍后再来看看”。
- [ ] 页面显示数据更新时间与“仅供娱乐参考”。
- [ ] 页面含 `noindex,nofollow,noarchive`。
- [ ] 不出现问号乱码、Emoji 转义文本、Tap to open、JSON 或内部路径。

## 公开祝福墙

- [ ] 不登录也可看到已有祝福。
- [ ] 可写昵称或匿名，提交后即时显示。
- [ ] HTML 和可点击链接被清洗。
- [ ] 同一 IP、同一生日页 10 分钟最多 3 条。
- [ ] Honeypot 有值时不创建真实留言。
- [ ] 刷新按钮可重新加载。
- [ ] “再看看更多祝福”只在确有下一页时出现。
- [ ] 管理员可在后台隐藏、恢复或删除留言。

## Supabase 与权限

- [ ] 新匿名用户可领取正确 token 的订单。
- [ ] 错误 token、过期 token、已被其他人领取的 token 都会失败。
- [ ] 用户 A 无法读取用户 B 的订单与私有照片。
- [ ] `birthday-order-private` 无公开 URL。
- [ ] 前端、GitHub、localStorage 中没有 service role 或第三方 API Key。
- [ ] `CLAIM_TOKEN_SECRET` 缺失时领取函数明确失败，不使用默认密钥。
- [ ] RLS 保持开启。
- [ ] 管理员登录后才可创建、审核、发布订单与导出配置。

## GitHub Pages

- [ ] `/`、`/admin.html`、`/birthday.html` 在线返回 200。
- [ ] 在 `/birthday-intake-system/` 子路径下，config、js、template previews 均无 404。
- [ ] 清除缓存或强制刷新后，资源版本号会加载新脚本。
- [ ] 手机 390 × 844 下没有横向裁切、重叠或无法点击的底部按钮。
