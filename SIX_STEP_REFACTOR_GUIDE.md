# 商家日常操作：从订单到生日页链接

这份是给商家自己用的，不需要写代码。

## 先理解两个链接

- **领取链接**：发给送礼人填写资料。格式包含 `?order=订单号&token=领取密钥`。
- **生日页链接**：审核并发布后生成。格式包含 `birthday.html?slug=...`。

领取链接只能让对应顾客填写自己的订单；生日页链接可以被送礼人转发，但不会被搜索引擎收录。

## 每一单怎么做

### 1. 商家创建订单

打开 `admin.html`，登录后创建订单：

1. 选择购买渠道。
2. 选择基础心意款 ¥16.6 或惊喜升级款 ¥28.8。
3. 选择领取链接有效期。
4. 创建后复制领取链接，发给顾客。

套餐在这里锁定。顾客只需要选择模板，不会看到重新买套餐的入口。

### 2. 顾客填写六步

顾客会依次完成订单确认、人物信息、照片、祝福、生日惊喜和提交确认。

他们看到的是“确认提交制作资料”，不会看到 JSON、数据库或后台术语。照片和草稿会保存到订单中，即使刷新也能继续填写。

### 3. 商家审核

顾客提交后，在后台打开订单详情：

- 查看封面、相册、重点图片和焦点位置。
- 查看主祝福、长祝福、署名。
- 检查已开启模块与惊喜盲盒意象。
- 需要补充时，把状态改为“需修改”，再联系顾客。
- 资料通过后改为“已批准”。

### 4. 发布生日页

订单处于“已批准”后点击“发布生日页面”。

后台会生成生日页链接。复制该链接发给送礼人或寿星即可。链接默认不允许搜索引擎收录，但可以转发。

### 5. 公开祝福墙

如果订单启用祝福墙，任何打开生日页的人都可匿名留言。管理员可以在订单详情中：

- 隐藏明显不合适的留言
- 恢复误隐藏留言
- 删除违规留言

系统已做自动清洗、链接拦截和频率限制，但不应该把私密信息写在公开祝福墙里。

## 今日好运与 API Key

今日好运会读取当天的星座运势和黄历宜忌。第一次上线前必须在 Supabase Edge Function Secrets 中填入 TianAPI 与聚合数据的 Key。Key 不要发送给顾客，也不要放进 GitHub。

如果服务暂时失败，生日页会如实显示“今日好运暂时在路上，请稍后再来看看”，不会编造内容。

## 有问题时先看这里

- 顾客打开后只能看到说明卡：检查链接是否同时含有 `order` 和 `token`。
- 顾客无法领取：检查 token 是否过期、订单是否已被别人领取、`CLAIM_TOKEN_SECRET` 是否已配置并重新部署 claim-order。
- 照片上传失败：确认 private bucket 存在、Storage RLS 没有被关闭、顾客正在使用自己的订单。
- 发布后打不开：检查 `get-published-page` 与 `publish-order` 是否已部署，`PUBLIC_BIRTHDAY_BASE_URL` 是否正确。
- 今日好运不可用：检查 `daily-luck` 已部署、两个第三方 Key 是否有效，等待缓存刷新后再测。
